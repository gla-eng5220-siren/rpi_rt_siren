# Building Instructions (on Debian Trixie)

```
sudo apt-get install cmake g++ pkg-config \
libjpeg-dev libv4l-dev libcatch2-dev \
libxnnpack-dev libpthreadpool-dev \
libavformat-dev libavcodec-dev libavutil-dev libswscale-dev \
libssl-dev libgpiod-dev libcamera-dev

mkdir -p build/release
cmake -DCMAKE_BUILD_TYPE=RelWithDebInfo -S . -B build/release
cmake --build build/release
```

# Minimal Example

The following command:

```
./build/release/flame_iris --mock-temp --alarm-stdout
```

is the minimal runnable example. It uses a mocked fake temperature sensor and prints alarm messages to stdout.

Although not very useful, it can show if your build works correctly.

# Vision Based Detection Setup

The following command:

```
./build/release/flame_iris \
  --libcamera 0 \
  --buzzer 26 \
  --model testdata/model \
  --webui-path webui
```

assuming

- A working camera, identifiable with `libcamera`, at index 0
- A GPIO-connected buzzer at pin 26

will perform vision-based detection and use the buzzer to generate alarm sounds.

# Advanced Usage

To make use of FlameIris, you need to specify one **sensor** and one **alarm**.

- Want to use a camera? Read [Camera Sensors](4.-Camera-Sensors)
- Want to use a temperature sensor? Read [Temperature Sensors](5.-Temperature-Sensors)
- Finally pick an alarm from [Alarming Methods](3.-Alarming-Methods)
