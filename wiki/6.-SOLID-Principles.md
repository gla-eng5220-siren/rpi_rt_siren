# Single Responsibility Principle (SRP)

## Frame Class Template

[Frame](https://gla-eng5220-siren.github.io/docs/classrpi__rt_1_1Frame.html) implements only the single responsibility -- serving as the container for NHCW data. Arbitrary functions such as JPEG decoding is offloaded to other functions.

## visual_classfying_model_t

[visual_classfying_model_t](https://gla-eng5220-siren.github.io/docs/classrpi__rt_1_1visual__classfying__model__t.html) holds no assumptions about HOW it is used, it only handle its single responsibility to run the model. The frame tracking, thresholding and other functions are wrapped in [visual_classify_logic_t](https://gla-eng5220-siren.github.io/docs/classrpi__rt_1_1visual__classify__logic__t.html). This improves reusability.

# Open/Closed Principle (OCP)

## detection_result_t

### Open

The [detection_result_t](https://gla-eng5220-siren.github.io/docs/classrpi__rt_1_1detection__result__t.html) is open for extension: it provides OOP virtual functions for overriding, to add more detailed reasoning for the detection.

### Close

The [detection_result_t](https://gla-eng5220-siren.github.io/docs/classrpi__rt_1_1detection__result__t.html) is closed for modification: once a detection result is reached, it becomes immutable internally, avoiding the alarming code to unexpectedly change the result.

# Liskov Substitution Principle (LSP)

## sensor_t

The [sensor_t](https://gla-eng5220-siren.github.io/docs/classrpi__rt_1_1sensor__t.html) has all sensors as its subclasses. When assembled in the main program, the subclasses can be replaced by each other without breaking behaviour.

# Interface Segregation Principle (ISP)

## factory methods like create_libcamera_sensor

Functions like [create_libcamera_sensor](https://gla-eng5220-siren.github.io/docs/group__Sensor.html#ga44b8ce8498b418872572f23aa232e960) segregates the implementation from interface definition. After creation, a polymorphism smart pointer is returned, without exposing the internals of the implementation to the user.
